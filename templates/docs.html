Detection Pipeline

1

Connection Capture

psutil.net_connections(kind='inet') scans all active TCP/UDP connections every 3 seconds. Each connection is mapped to its owning process via PID.

2

Process Enrichment

For each PID, we extract: process name, executable path, memory usage, CPU %. We check the path against known-safe lists and flag temp-directory executables.

3

Trust Scoring

Each process receives a 0-100 trust score based on: identity verification, destination IP reputation, suspicious port usage, data volume anomalies, and connection frequency.

4

Alert Generation

Processes with trust scores below 70 generate explainable alerts with severity levels, human-readable reasons, and suggested remediation actions.

Trust Score Engine

Every process starts at 100 and loses points based on:

Factor Penalty Example
Unknown process name−20exfil_svc.exe
Running from %TEMP%−25C:\Users\Temp\payload.exe

Suspicious port−15Port 4444, 1337
High data volume (>100MB)−10Bulk outbound transfer
High connection count (>20)−10Port scanning pattern

Mitigation Actions

cancel

Terminate Process

Kills the process via psutil.Process.terminate()

API: POST /api/action/terminate

block

Block IP

Adds the IP to an in-memory blocklist. Blocked IPs are excluded from connection results.

API: POST /api/action/block-ip

check_circle

Mark Safe

Whitelists a process PID and removes its alert from the active list.

API: POST /api/action/mark-safe
